Architecture Hexagonale

1-2 jours • Junior à confirmé

Formation pratique pour concevoir des logiciels testables, maintenables et évolutifs.

Le problème

Code difficile à tester

Logique métier couplée avec frameworks et BDD.

Dépendances mêlées

Domaine dépend de détails techniques.

Dette technique

Les assistants IA amplifient le problème sans architecture solide.

La solution

Domaine isolé

Logique métier pure, sans dépendance technique.

Interchangeabilité

Changez une technologie sans toucher au métier.

Testabilité

Tests unitaires rapides sans infrastructure.

Objectifs

Vous serez capable de :

  • Concevoir une architecture hexagonale
  • Séparer domaine et technique
  • Définir ports et adaptateurs
  • Rendre le code 100% testable
  • Refactorer une appli existante
  • Utiliser les assistants IA efficacement

Public

Développeurs (junior à confirmé), architectes, tech leads

Python ou C++, bases OOP, expérience projet

Tests unitaires, intégration, mocks, coverage

Module 5 : Refactoring
